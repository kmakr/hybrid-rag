🔍 Query Pipeline
1
Load Document
chunker.py
Files are read from data/. PDFs are extracted page-by-page via pdfplumber, DOCX via docx2txt, and plain text files are read directly.
Input
data/hkjc_rules_2024.pdf (42 pages)
↓
full text string
2
Chunk Document
chunker.py
The full text is split using RecursiveCharacterTextSplitter with chunk_size=500 chars and overlap=50 chars. Each chunk gets a zero-based chunk_index within its source document.
Output — 3 of 87 chunks

'Horse' means any horse registered with the Club. 'Race Meeting' means any race meeting..." chunk #2 "...approved by the Stewards. An Owner must ensure their horse is properly identified before..."
↓
87 raw chunks
3
Contextualize Chunks
contextualizer.py
Each chunk is sent to MiniMax M2.7 via OpenRouter along with the full parent document. The model generates a short situating description. Runs in parallel across 4 threads to speed up ingestion.
Prompt sent to MiniMax M2.7
<document> Chapter 1 – Definitions. In these Rules... [full 42-page document] </document> Here is the chunk we want to situate: <chunk> An Owner must ensure their horse is properly identified before... </chunk> Please give a short succinct context to situate this chunk...
Model response (context)
"This chunk is from Chapter 3 of the HKJC Rules of Racing, covering owner obligations regarding horse identification and registration requirements prior to race entry."
full_text stored in DB (content + context)
An Owner must ensure their horse is properly identified before... This chunk is from Chapter 3 of the HKJC Rules of Racing, covering owner obligations regarding horse identification and registration requirements prior to race entry.
↓
87 enriched chunks
4
Generate Embeddings
embedder.py
The full_text (content + context) of every chunk is encoded locally using all-MiniLM-L6-v2 via sentence-transformers. Produces a 384-dimensional float vector. No API call — runs on your machine.
Output
chunk #2 → [0.0231, -0.1847, 0.0563, 0.2901, ..., -0.0412] (384 floats)
↓
vectors + metadata
5
Store in PostgreSQL
db.py
Each chunk is inserted into the chunks table. PostgreSQL automatically builds a tsvector column for full-text search. Two indexes are maintained: ivfflat for vector search and GIN for full-text.
Row stored in chunks table

1
User Question
api.py / query.py
Question arrives from the chat UI (POST /chat) or the CLI script. Parameters k and rerank control retrieval depth and reranking.
Request
question: "What are the identification requirements for horse owners?" k: 5, rerank: true
↓
embed question
2
Embed Query
embedder.py
The question is encoded with the same all-MiniLM-L6-v2 model used at ingest time, producing a 384-dim query vector.
Output
query vector → [0.0819, -0.2341, 0.1102, ...] (384 floats)
↓
parallel search
Dense Search (pgvector)
3a
Vector Search
db.py
Cosine similarity between query vector and all stored embeddings. Retrieves top-150 candidates.
SQL
ORDER BY embedding <=> query_vec LIMIT 150
Sparse Search (tsvector)
3b
Full-Text Search
db.py
BM25-style keyword match using Postgres tsvector GIN index. Retrieves top-150 candidates.
SQL
WHERE tsv @@ plainto_tsquery('identification requirements owner') ORDER BY ts_rank DESC LIMIT 150
↘        ↙
4
Reciprocal Rank Fusion
retriever.py
Results from both searches are merged. Each chunk gets a fused score: 0.8 × (1/(60+rank_dense)) + 0.2 × (1/(60+rank_sparse)). Since rerank=true, top 50 are kept (k×10).

↓
50 candidates → reranker
5
Cross-Encoder Rerank
reranker.py
(--rerank only)
Each of the 50 candidates is scored against the query using ms-marco-MiniLM-L-6-v2 — a cross-encoder that reads both query and chunk together. More accurate than embedding similarity. Top-5 are kept.

↓
top 5 chunks as context
6
Generate Answer
generator.py
The top-5 chunks are passed as context to MiniMax M2.7 via OpenRouter. The model is instructed to answer only from the provided context and say so if it can't.
Prompt (simplified)
<context> An Owner must ensure their horse is properly identified before... --- The Stewards may at any time require proof of identity... --- All horses must carry a microchip implanted by an approved vet... </context> Question: What are the identification requirements for horse owners?
